# 기획서 — Editor Startup Profiler

> 작성일: 2026-04-25  
> 카테고리: 에디터 UX / 생산성  
> 우선순위: 중

---

## 개요

UE5 에디터 시작 시 각 단계(플러그인 로드, 모듈 초기화, 에셋 레지스트리 빌드 등)의 소요 시간을 측정하고, 병목 구간을 리포트하는 에디터 유틸리티.

---

## 문제 정의

- 프로젝트 규모가 커질수록 에디터 시작 시간이 늘어나지만 어떤 플러그인/모듈이 원인인지 파악하기 어렵다.
- 개발자마다 설치한 플러그인이 달라 재현이 힘들다.
- 매번 수동으로 로그를 뒤져야 한다.

---

## 목표

- 에디터 시작 시간을 단계별로 측정
- 느린 구간 상위 N개를 자동으로 지목
- 결과를 리포트 파일로 저장

---

## 주요 기능

| 기능 | 설명 |
|---|---|
| 단계별 타이밍 수집 | 플러그인 로드 / 모듈 Init / 에셋 레지스트리 / 셰이더 컴파일 구간 측정 |
| 상위 병목 리포트 | 소요 시간 상위 10개 항목 표시 |
| 비교 모드 | 이전 실행 결과와 diff — 추가된 플러그인이 얼마나 영향을 주는지 확인 |
| CSV / JSON 출력 | CI 파이프라인 연동 가능한 구조화 출력 |

---

## 구현 방향

- `FEngineLoop` 및 `IModuleInterface::StartupModule()` 호출 타이밍 후킹
- 언리얼 내장 `GLog` 파싱 또는 `-logcmds` 플래그 활용
- Editor Utility Widget으로 결과 시각화 (바 차트)
- 출력 경로: `Saved/Profiling/StartupProfile_YYYYMMDD.json`

---

## 입출력

**입력**
- 없음 (에디터 시작 시 자동 실행) 또는 메뉴에서 수동 트리거

**출력**
- `Saved/Profiling/StartupProfile_YYYYMMDD.json`
- 에디터 내 결과 패널 (선택)

---

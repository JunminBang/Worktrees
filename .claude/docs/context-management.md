# 컨텍스트 관리

컨텍스트는 Claude Code 세션에서 가장 중요한 자원이다. 능동적으로 관리할 것.

## 파일 기반 상태 관리 (핵심 전략)

**파일이 기억이다 — 대화가 아니다.** 대화는 압축되거나 사라진다.
디스크의 파일은 압축과 세션 종료를 거쳐도 유지된다.

### 세션 상태 파일

`production/session-state/active.md`를 살아있는 체크포인트로 유지한다.
주요 마일스톤 이후 갱신:

- 설계 섹션 승인 후 파일에 저장
- 아키텍처 결정 완료
- 구현 마일스톤 달성
- 테스트 결과 확보

상태 파일 포함 내용: 현재 태스크, 진행 체크리스트, 핵심 결정 사항, 작업 중인 파일, 미해결 질문.

### 상태 라인 블록 (프로덕션 이후 단계)

프로젝트가 Production, Polish, Release 단계에 있을 때,
상태 라인 스크립트가 파싱할 수 있는 구조화된 블록을 `active.md`에 포함:

```markdown
<!-- STATUS -->
Epic: 전투 시스템
Feature: 근접 전투
Task: 히트박스 감지 구현
<!-- /STATUS -->
```

- 세 필드(Epic, Feature, Task) 모두 선택 사항 — 해당되는 것만 포함
- 작업 영역 전환 시 블록 갱신
- 상태 라인은 `전투 시스템 > 근접 전투 > 히트박스` 형태의 브레드크럼으로 표시
- 활성 작업 없을 때 블록 제거 또는 비움

중단 발생 시(압축, 충돌, `/clear`) 상태 파일을 먼저 읽는다.

### 점진적 파일 작성

다중 섹션 문서(설계 문서, 아키텍처 문서, 로어 항목) 작성 시:

1. 즉시 파일 생성 — 스켈레톤(모든 섹션 헤더, 빈 본문)
2. 대화에서 섹션 하나씩 논의하고 초안 작성
3. 승인된 섹션은 즉시 파일에 저장
4. 각 섹션 작성 후 세션 상태 파일 갱신
5. 섹션 저장 후 해당 섹션의 이전 대화는 안전하게 압축 가능 — 결정 사항이 파일에 있기 때문

이렇게 하면 컨텍스트 윈도우가 문서 전체 대화 이력(~30-50k 토큰)이 아닌
현재 섹션 논의(~3-5k 토큰)만 보유하게 된다.

## 선제적 압축

- 컨텍스트 사용량 ~60-70%에서 **선제적으로 압축** — 한계에 다다른 후 대응 금지
- 무관한 태스크 사이, 또는 2회 이상 수정 시도 실패 후 **`/clear` 사용**
- **자연스러운 압축 포인트:** 섹션을 파일에 저장한 후, 커밋 후, 태스크 완료 후, 새 주제 시작 전
- **집중 압축:** `/compact Focus on [현재 태스크] — 섹션 1-3은 파일에 저장, 섹션 4 작업 중`

## 태스크 유형별 컨텍스트 예산

- 가벼움 (읽기/검토): ~3k 토큰 시작
- 중간 (기능 구현): ~8k 토큰
- 무거움 (다중 시스템 리팩토링): ~15k 토큰

## 서브에이전트 위임

메인 세션을 깔끔하게 유지하기 위해 서브에이전트를 활용한다.
서브에이전트는 자체 컨텍스트 윈도우에서 실행되고 요약만 반환한다:

- **서브에이전트 사용**: 여러 파일 탐색, 낯선 코드 조사,
  5k 토큰 이상의 파일 읽기가 필요한 리서치
- **직접 읽기 사용**: 확인할 파일 1-2개를 정확히 알고 있을 때
- 서브에이전트는 대화 이력을 상속받지 않음 — 프롬프트에 전체 컨텍스트 제공 필요

## 압축 지시 사항

컨텍스트 압축 시 다음 내용을 요약에 보존:

- `production/session-state/active.md` 참조 (상태 복구를 위해 먼저 읽기)
- 이번 세션에서 수정한 파일 목록과 목적
- 내린 아키텍처 결정 및 근거
- 활성 스프린트 태스크와 현재 상태
- 에이전트 호출 결과 (성공/실패/차단)
- 테스트 결과 (통과/실패 수, 구체적 실패 항목)
- 미해결 차단 요소 또는 사용자 입력 대기 중인 질문
- 현재 태스크와 진행 단계
- 현재 문서에서 파일에 저장된 섹션과 진행 중인 섹션

**압축 후:** `production/session-state/active.md`와 작업 중인 파일을 읽어 컨텍스트 복구.
파일에 결정 사항이 있다; 대화 이력은 부차적이다.

## 세션 충돌 후 복구

세션이 죽거나("prompt too long") 작업 계속을 위해 새 세션을 시작할 때:

1. `session-start.sh` 훅이 자동으로 `active.md`를 감지하고 미리보기
2. 컨텍스트를 위해 전체 상태 파일 읽기
3. 상태에 나열된 부분적으로 완성된 파일 읽기
4. 다음 미완성 섹션 또는 태스크에서 계속
